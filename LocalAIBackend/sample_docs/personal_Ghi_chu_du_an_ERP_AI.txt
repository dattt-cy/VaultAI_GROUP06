1. TỔNG QUAN DỰ ÁN

Dự án tích hợp module AI vào hệ thống ERP hiện có. Mục tiêu: tự động phân tích dị thường trong chi tiêu, dự báo dòng tiền 30 ngày, và đề xuất tối ưu hóa ngân sách.

Tech stack quyết định:
- Backend AI: Python FastAPI + LangChain
- Model: Fine-tuned LLM (bắt đầu với GPT-4o API, sau chuyển sang mô hình local)
- Vector DB: ChromaDB (self-hosted)
- Integration: REST API với ERP backend (Java Spring Boot)
- Logging/Monitoring: ELK Stack

3. VẤN ĐỀ KỸ THUẬT ĐÃ GẶP VÀ CÁCH GIẢI QUYẾT

Vấn đề 1: Dữ liệu chi tiêu bị imbalanced (chi tiêu bất thường chỉ 2% tổng records)
Giải pháp: SMOTE oversampling + điều chỉnh class weights trong model training.
Kết quả: F1-score tăng từ 0.61 lên 0.82.

Vấn đề 2: Latency cao khi gọi GPT-4o API (~3-5 giây/request)
Giải pháp: Caching kết quả tương tự bằng Redis (semantic cache với cosine similarity > 0.92).
Kết quả: 68% requests được serve từ cache, latency giảm xuống <200ms.

Vấn đề 3: ERP API rate limiting (100 requests/phút)
Giải pháp: Implement queue với Celery + Redis, batch processing theo giờ thay vì real-time.
